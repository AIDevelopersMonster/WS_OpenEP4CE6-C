# 8 Push Buttons Demo

## Описание проекта

Этот проект реализует управление светодиодами с помощью 8 кнопок. Каждое нажатие кнопки изменяет состояние светодиодов. Используется система с 8 кнопками (key) и 4 светодиодами (led). Проект предназначен для работы с FPGA или CPLD и использует язык описания аппаратуры Verilog.

### Модуль `joystick`

Модуль `joystick` реализует управление светодиодами на основе состояния кнопок. Каждое нажатие кнопки изменяет состояние светодиодов с задержкой, контролируемой тактовым сигналом и счетчиком.

- **Входы:**
  - `clk`: Тактовый сигнал.
  - `reset`: Сигнал сброса.
  - `key`: 8 кнопок (16-битный вход, где каждый бит представляет кнопку).
  
- **Выходы:**
  - `led`: 4 светодиода, которые изменяются в зависимости от нажатых кнопок.

### Описание работы

- Когда нажимается кнопка, состояние светодиодов меняется.
- Кнопки представлены в двоичной форме: каждая кнопка соответствует одному биту в 8-битном числе.
- Используется задержка между сменой состояний светодиодов.

## Описание файлов проекта

### 1. Файл с расширением `.v` (Verilog)

Файл с расширением `.v` — это исходный код на языке Verilog. В нашем случае это файл с описанием модуля `joystick`, который управляет светодиодами. Он включает описание всех логических элементов и процессов, таких как счетчик для генерации задержки и обработчик состояния кнопок.

Пример:
- `joystick.v` — исходный код модуля, который управляет светодиодами в зависимости от состояния кнопок.

### 2. Файл с расширением `.qpf` (Quartus Prime Project File)

Файл `.qpf` (Quartus Prime Project File) — это главный файл проекта в среде разработки Intel Quartus Prime. Он содержит информацию о проекте, такую как имя проекта, настройки синтеза и другие параметры. Этот файл создается и используется при сборке и компиляции проекта в Quartus.

Пример:
- `project.qpf` — главный файл проекта, содержащий информацию о проекте и его параметрах.

### 3. Файл с расширением `.qsf` (Quartus Settings File)

Файл `.qsf` (Quartus Settings File) — это файл настроек проекта для Quartus Prime. Он хранит информацию о пинах, портах, констрейтах и других специфических настройках устройства. Этот файл необходим для правильной компиляции проекта и корректного отображения пинов и портов на FPGA-плате.

Пример:
- `project.qsf` — файл, который содержит информацию о пинах и констрейтах проекта. Например, он указывает, какие порты соответствуют кнопкам и светодиодам.

### 4. Файл с расширением `.sof` (SRAM Object File)

Файл `.sof` (SRAM Object File) — это файл, который генерируется после компиляции проекта в Quartus Prime. Этот файл содержит битовый поток для загрузки в SRAM FPGA. Он используется для программирования FPGA устройства с уже скомпилированным проектом.

Пример:
- `project.sof` — файл, который загружается на FPGA для конфигурации устройства.

### 5. Файл с расширением `.pof` (Programmer Object File)

Файл `.pof` (Programmer Object File) — это файл, который используется для программирования конфигурации на устройство, которое использует Flash-память для хранения настроек. В отличие от файла `.sof`, который загружается в SRAM, `.pof` сохраняет конфигурацию в Flash-память FPGA.

Пример:
- `project.pof` — файл, который используется для загрузки конфигурации в Flash-память устройства.

### 6. Файл с расширением `.jdi` (JTAG Debug Information File)

Файл `.jdi` (JTAG Debug Information File) — это файл, который используется для отладки проекта через интерфейс JTAG. Этот файл содержит информацию, необходимую для отладки на уровне аппаратного обеспечения. Обычно он генерируется вместе с проектом для облегчения отладки на реальном устройстве.

Пример:
- `project.jdi` — файл, который используется для отладки и мониторинга состояния FPGA с помощью JTAG интерфейса.

## Инструкция по сборке

1. Откройте проект в Intel Quartus Prime, используя файл проекта `.qpf`.
2. Убедитесь, что все пины правильно назначены в файле `.qsf`.
3. Скомпилируйте проект, чтобы сгенерировать файл `.sof` для загрузки на FPGA.
4. Используйте файл `.pof` для программирования устройства с использованием Flash-памяти (если требуется).
5. Для отладки используйте файл `.jdi` с JTAG интерфейсом.
